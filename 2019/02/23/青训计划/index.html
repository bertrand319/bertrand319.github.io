项目目标

  1. 帮助工程师技能转型
  2. 提升骨干员工核心竞争力

技能转型实施路径

一期目标帮助前端工程师快速转型到后端工程师

为了保证目标实现，需要做到几点：

  1. 明确的任务项及时间节点，需要产出知识分布的思维导图
  2. 每个任务项有明确验收标准，需要设立审核人员
  3. 评估合适转型的人员，不能一棒子，需要确认评估标准
  4. 在绩效考核上需要做出适当的倾斜

为实现该目标，涉及到以下配合角色

  1. 任务编写者，需要安排团队资深人员进行编写，可以考虑一下近期离职员工，结合他们的交接计划做好这项工作。
  2. 审核者，通过简单面谈审核的方式去了解转型人员是否达到目标，可以简单设立一下面谈题目的方式。
  3. 能力评估者，通过360度访谈的方式来进行了解人员素质。
